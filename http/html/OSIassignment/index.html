Overview
  • Logical Addressing and Path Determination
  • The network layer provides the functional and procedural means of transferring variable length data sequences (called datagrams) from one node to another connected in different networks.
  • A network is a medium to which many nodes can be connected, on which every node has an address and which permits nodes connected to it to transfer messages to other nodes connected to it by merely providing the content of a message and the address of the destination node and letting the network find the way to deliver the message to the destination node, possibly routing it through intermediate nodes.
  • Large messages are often split into smaller pieces and sent seperately then reassembled at a later node.
  • A number of layer-management protocols, a function defined in the management annex, ISO 7498/4, belong to the network layer. These include routing protocols, multicast group management, network-layer information and error, and network-layer address assignment.
IPv4
  • Internet Protocal Version 4
  • Deployed in 1981
  • Can be configured manually and by using dhcp
  • Uses both broadcast and multicast
  • Addresses are made up of 32 bits allowing for a maximum of 232 or 4,294,967,296 unique addresses (a limit currently reached)
  • Often expressed in quad-dot format created by grouping the bits into 4 8-bit segments which are, in turn, displayed as decimal integers seperated by dots (e.g. 1100 0000 1011 0111 0000 0010 0000 1001 is converted to 192.183.2.9)
  • This is currently the most used internet protocol.
IPv5
  • IPv5 was a flop.
  • Had minor improvements over IPv4 but all the problems
  • Officially called Internet Stream Protocol (ST and later ST-II)
  • Never transitioned to public use.
IPv6
  • Deployed in 1999
  • Can be configured manually, with Universal Plug and Play (UPnP), or with DHCP
  • Has no broadcast and contains various forms of multicast
  • Better bandwidth efficiency
  • Lowers operations expenses and reduces error with regard to configuration
  • Addresses are made up of 128 bits allowing for a maximum of 2128 or 340,282,366,920,938,463,463,374,607,431,768,211,456 (3.4×1038 or 340 Undecillion) unique addresses (a very big number though admittedly less than the number atoms in the Earth that being somewhere between 1049 and 1050)
  • Often expressed as four hextets created by grouping the bits into 8 16-bit segments which are, in turn, displayed as hexadecimal integers seperated by colons (e.g. 0010 0110 0000 0000 0110 1100 0101 0100 0111 1000 0000 0000 0000 1000 1110 1001 0001 0000 0101 1110 0100 1111 1100 0101 0110 1010 1011 0000 0000 1100 0010 1111 is converted to 2600:6c54:7800:08e9:105e:4fc5:6ab0:c2f)
  • This is currently the newest internet protocol.
